# Rotas aninhadas

O modelo de blog suporta postagens em subpastas aninhadas. Isso ajuda na organização e pode ser usado para agrupar postagens de conteúdo semelhante, por exemplo, uma série em várias partes. Este artigo é um exemplo de rota aninhada! Ele está localizado na pasta `/data/blog/nested-route`.

## Como?

Simplifique criando várias pastas na pasta principal `/data/blog` e adicione seus arquivos `.md`/`.mdx` nelas. Você pode até criar algo como `/data/blog/nested-route/deeply-nested-route/my-post.md`

Usamos rotas catch all do Next.js para lidar com a criação de rotas e caminhos.

## Casos de uso

Aqui estão algumas razões para usar rotas aninhadas:

- Organização de conteúdo mais lógica (os blogs sempre serão exibidos com base na data de criação)
- Artigos em várias partes
- Diferentes sub-rotas para cada autor
- Internacionalização (embora seja recomendável usar o [roteamento i8n integrado do Next.js](https://nextjs.org/docs/advanced-features/i18n-routing))

## Nota

- Os links de postagens anteriores/próximas na parte inferior do modelo são atualmente classificados por data. Pode-se considerar modificar o modelo para direcionar o leitor para a postagem anterior/próxima da série, em vez de por data.
